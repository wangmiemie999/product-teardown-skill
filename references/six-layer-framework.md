# 六层分析框架

## 1. 市场层

分析市场定义、细分赛道、需求驱动因素、发展阶段、地域差异、直接与间接竞争者、替代方案、定位、进入壁垒、扩张空间及政策或平台风险。用真实替代行为校验竞争边界。

## 2. 商业层

区分用户、客户、购买者和预算持有人。分析收费模式、价格结构、获客渠道、销售方式、收入来源、主要成本、毛利潜力、留存、扩张收入、单位经济性、网络效应以及对供应商或平台的依赖。

对 AI 产品额外检查推理成本、免费额度滥用、模型供应商议价权，以及产品价值增长能否快于模型成本。

## 3. 用户层

识别核心角色、行为特征、触发场景、Jobs to Be Done、原有替代方案、核心旅程、首次价值时间、高低频任务、留存与流失因素，以及学习、迁移、信任和隐私成本。区分用户陈述、可观察行为与分析推断。

## 4. 技术层

分析客户端、服务端、基础设施、核心模块、系统边界、API、插件、生态集成、性能、可靠性、扩展性、安全、权限、隐私、技术债务、第三方依赖以及自研、采购、合作的边界。只有存在证据时才判断具体技术栈。

## 5. 模型层

适用于 AI 或算法驱动产品。分析模型类型与供应商、单模型或多模型策略、路由、Prompt、RAG、微调、Agent、上下文管理、工具调用、输出约束、评测体系、安全机制、失败处理、人工介入，以及质量、延迟与成本权衡。

重点判断产品价值主要来自基础模型还是产品系统；基础模型升级后壁垒会增强、迁移还是消失。

## 6. 数据基础层

分析数据来源、所有权、授权、采集、清洗、标注、存储、检索、索引、更新、质量、时效、覆盖率、隔离、隐私、合规、反馈闭环、评测数据、冷启动和数据网络效应。

验证“使用越多产品越好”是否真实成立，以及数据优势是否独占、合法、稳定且难以复制。

## 层间关系

至少绘制一条完整因果链，例如：

`目标市场 → 用户任务 → 产品体验 → 技术/模型能力 → 数据反馈 → 体验改进 → 留存/收入 → 更多数据`

同时寻找断裂点：市场与用户错配、价值与定价错配、体验与技术能力错配、模型成本与收入错配、数据获取与合规错配。
